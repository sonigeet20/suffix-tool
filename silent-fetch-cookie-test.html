Cookies After Fetch

How to Verify

Steps:
  1. Open DevTools: Press F12 (Windows/Linux) or Cmd+Option+I (Mac)
  2. Go to Network Tab: Watch for the fetch request to your tracking URL
  3. Check Request Headers: Should include "Cookie: ..." if credentials are being sent
  4. Check Response Headers: Should have "Set-Cookie: ..." if server sets new cookies
  5. Check this page's cookies: Click "Refresh Cookies" button below to see all stored cookies
  6. CORS Issues: If you see CORS errors, the server may need to set proper headers

Server Requirements for Cookies

For cross-domain cookies to work, the tracking server MUST respond with:
Access-Control-Allow-Origin: [your-domain]
Access-Control-Allow-Credentials: true
Set-Cookie: [name]=[value]; SameSite=None; Secure
